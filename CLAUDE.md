## Project Overview

This is the source repository for the **HoTT Book** (*Homotopy Type Theory: Univalent Foundations of Mathematics*), a LaTeX textbook produced at the Institute for Advanced Study in 2012/13. Licensed under CC BY-SA 3.0.

## Build Commands

Requires TeX Live 2012+ and GNU Make. `version.tex` is auto-generated by Make.

```bash
make                          # Default: hott-online.pdf (color, screen reading)
make hott-online.pdf          # Color version for online reading
make hott-ebook.pdf           # E-book format (small margins)
make hott-letter.pdf          # B&W letter paper for printing
make hott-a4.pdf              # B&W A4 paper for printing
make hott-ustrade.pdf         # US Trade format (Lulu binding)
make hott-arxiv.pdf           # arXiv submission version
make exercise_solutions.pdf   # Exercise solutions
make errata.pdf               # First edition errata
make all                      # All formats + covers
make clean                    # Remove build artifacts
make labelcheck               # Verify theorem/section numbering consistency
```

## Architecture

### Book chapters (in dependency/reading order)

`main.tex` is the master document; top-level `hott-*.tex` files are format-specific wrappers that `\input{main.tex}` with different option files (`opt-*.tex`).

Content chapters: `introduction.tex` → `preliminaries.tex` → `basics.tex` → `logic.tex` → `equivalences.tex` → `induction.tex` → `hits.tex` → `hlevels.tex` → `homotopy.tex` → `categories.tex` → `setmath.tex` → `reals.tex` → `formal.tex`

### Macro system (`macros.tex`)

All mathematical notation is defined via custom macros. Key conventions (see `CONVENTIONS.txt` for full list):

- `\jdeq` — judgmental/definitional equality (≡); `\defeq` — definition (:=)
- `\refl{x}`, `p \ct q` (path concatenation), `\opp{p}` (path inverse)
- `\trans{p}{x}` (transport), `\map{f}{p}` (ap), `\mapdep{f}{p}` (apd)
- `\eqv{A}{B}` (equivalence type), `\type`, `\set`, `\prop` (universes)
- `\prd{x:A} B(x)` (dependent product), `\sm{x:A} B(x)` (dependent sum)
- Theorem environments: `\thm`, `\cor`, `\lem`, `\defn`, `\rmk`, `\eg`, `\ex`
- Use `\autoref{label}` for cross-references (auto-generates "Theorem 3.1" etc.)
- Labels: only letters, digits, colon `:`, and dash `-`

### CI/CD (`.github/workflows/ci.yml`)

Runs on `danteev/texlive:2025-07-15`. Checks em-dash spacing (`[^ ]---[^ ]`), builds PDFs, auto-marks errata, generates nightly builds to GitHub Pages/wiki.

### Coq companion (`coq_introduction/`)

Separate Makefile; builds `Reading_HoTT_in_Coq.v` into HTML (coqdoc) and PDF.

### Obsidian vault (`hott-vault/`)

Knowledge base of markdown notes organized by chapter topics (Type Theory, HoTT Basics, Equivalences, HITs, Categories, etc.).

## Contribution conventions

- Chapters end with unnumbered "Notes" (citations/references) and "Exercises" sections
- Citations go in `references.bib`; use `\cite` in text
- Changes affecting theorem numbering must target future editions (see `CONTRIBUTING.md`)
- `make labelcheck` compares against `main.labelnumbers.first-edition`
